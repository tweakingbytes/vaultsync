Troubleshooting

Secure File System

Files are stored on the external drive in a folder called VaultSync. The location is usually in the same folder as the VaultSync program file.

The vault contains an index, a control file and folders containing the files. The folders help speed up file management on flash drives that don't perform well with lots of files in a single folder. The folders and the files are named anonomously.

Getting out of trouble

  • VaultSync has a lot of error checking code but sometimes things happen that it can't control. VaultSync will always try to protect the current vault in the event of a problem and tell you what has happened. Often the problem is temporary and is resolved by running VaultSync again.
  • The index is encrypted but the last working version is always kept in case something unexpected happens. In the unlikely event that the index is unable to be opened then remove the corrupted version index.dat and rename index.bak as index.dat. A corrupted index is usually noticible from Windows Explorer becasue the file size is very different to the backup.
  • If the control.dat file is missing then the vault has been corrupted and cannot be recovered. Be very careful not to accidentially delete this file.
  • Synchronised offline file systems like OneDrive, etc. use hidden control files that may be temporarily locked when VaultSync is synchronising. The reported error can safely be ignored. The file can also be added to the Ignore list so it isn't synchronised next time. You need to be careful with the Ignore pattern so that other files are not accidentially also ignored.
